# Лабораторная работа №4

## Тема
Графический пользовательский интерфейс для модели готовки яичницы из лабораторной работы №1.

## Архитектура
В проекте используется шаблон MVC.

- `model` содержит общую бизнес-логику и состояние предметной области.
- `view` содержит CLI-представление и GUI-представление на PyQt.
- `controller` содержит контроллеры для CLI и GUI.

Общий код модели используется и в CLI, и в GUI. За счет этого поведение приложения одинаковое в обоих режимах запуска.

## Предметная область
Программа моделирует процесс приготовления яичницы. В модели используются обязательные сущности:

- яйца
- сковорода
- масло
- приправы
- плита
- лопатка

## Возможности приложения
- Разбить яйца с накоплением количества, максимум 10 штук.
- Включить электроплиту, выбрать мощность конфорки и время готовки.
- Добавить масло: сливочное или подсолнечное.
- Обжарить выбранное количество яиц из уже разбитых.
- Добавить приправы: соль, перец, паприка, универсальная.
- Перемешать и подать блюдо.
- Посмотреть текущее состояние модели.

## Правила приготовления
- Мощность 1: 15-20 минут
- Мощность 2: 11-14 минут
- Мощность 3: 7-10 минут
- Мощность 4: 5-6 минут
- Мощность 5: 3-4 минуты
- Мощность 6: 1-2 минуты

Если время меньше допустимого диапазона, при подаче выводится, что яйца получились сырые. Если время больше диапазона, при подаче выводится, что яйца сгорели. В обоих случаях указывается правильный диапазон времени.

## Структура проекта
- `main.py` — точка входа.
- `model/cooking_model.py` — общая модель приложения.
- `model/entities.py` — доменные сущности.
- `model/exceptions.py` — исключения модели.
- `view/cli_view.py` — представление для CLI.
- `view/gui_view.py` — представление для GUI на PyQt.
- `controller/cli_controller.py` — контроллер CLI.
- `controller/gui_controller.py` — контроллер GUI.
- `tests/test_cooking.py` — unit-тесты.

## Запуск приложения
CLI:

```bash
python main.py
```

GUI:

```bash
python main.py --gui
```

Если для GUI не установлен `PyQt5` или `PyQt6`, приложение завершится с понятным сообщением.

## Запуск тестов
```bash
python -m unittest discover -s tests -v
```

## Тестирование
Покрыты тестами:

- основные сценарии работы модели
- обработка ошибочных ситуаций
- ограничение на максимальное количество яиц
- проверка сырых и сгоревших яиц
- формирование состояния для CLI-контроллера
